# FLOW-06: 打磨上线

## 目标
响应式适配、PWA支持、部署上线。

## 前置条件
- FLOW-01 到 FLOW-05 全部完成

## 实现步骤

### Step 1: 响应式适配
- 手机端 375px 优先
- 输入框和按钮适配触摸操作
- 结果区域在小屏上纵向排列
- 表格横向可滚动

### Step 2: PWA 支持
- 创建 manifest.json（应用名称、图标、主题色）
- 注册 Service Worker（缓存静态资源，离线可用）
- 添加"添加到桌面"提示

### Step 3: SEO 基础
- 页面标题包含关键词："公差查询 | 公差配合 | ISO 286 | GB/T 1800"
- meta description 包含核心功能描述
- 页面包含 h1 标题和结构化内容（便于百度收录）

### Step 4: 性能优化
- JSON 数据预加载
- 零外部依赖（不用CDN）
- 总体积极小（目标 < 100KB gzip）

### Step 5: 部署
- Vercel 或 Cloudflare Pages（免费）
- 自定义域名（可选）
- HTTPS（自动）

### Step 6: 访问统计
- 轻量统计（umami 或 百度统计）
- 追踪：PV、功能使用频率

## 测试标准
- [ ] 手机 Chrome 打开正常
- [ ] 离线模式下基础查询可用（PWA）
- [ ] Lighthouse 评分 > 90
- [ ] 百度可收录（SSR 或预渲染非必需，静态HTML即可）
